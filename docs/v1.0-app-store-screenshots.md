# v1.0 App Store スクリーンショット計画

App Store Connect 申告（v1.0 Bare Seed）向けのスクリーンショット仕様。
**最重要：Before / After**（spec §16.3）。**Timelapse は v1.0 から全プラン無料**（`ReleaseConfig.timelapseEnabled = true`）なので提出物に含めてよい。**Search は v1.0 で封鎖**（`searchEnabled = false`）のため登場させない。

---

## 0. 方針サマリ

| 項目 | 値 |
|---|---|
| ASO ポジション | 「植物管理」ではなく **「植物の時間を残す記録」**（spec §15.1 / §16.1） |
| 訴求順 | Before/After → Compare 指標 → Timeline → Observation → Quick Log → Collection |
| デザイン基調 | Dark / Quiet Luxury。UI kit の `colors_and_type.css` トークン |
| 言語 | 日本語（JP） |
| 出力 | PNG（6.9" iPhone は必須、6.5" もアップロード） |

### 必須サイズ（App Store Connect, 2026 時点）

| デバイス区分 | 想定機種 | 解像度（px, 縦） |
|---|---|---|
| 6.9" iPhone（必須） | iPhone 16 Pro Max | **1290 × 2796** |
| 6.5" iPhone（互換） | iPhone 11 Pro Max | 1242 × 2688 |

> 6.7"（1290×2796）と 6.9" は同じ解像度を受け付ける。6.5" は別レンディションで作成。

### 守ること（v1.0 制約）

- 全文検索、タグ、iCloud バックアップは **画面・コピー双方に出さない**（v1.0.1 Pack で開放）。タイムラプスは v1.0 から無料のため出してよい。
- 「クラウド」「同期」「アカウント」も登場禁止（v1.0 は端末内完結）。
- 「水やり通知」は v1.0 で出荷済みだが、ASO の主張は **観測** 軸。通知導線はサブで触れる程度に留める。

## 1. フレーム構成（6 枚）

順序は左から右にスクロールされる前提で、最初の 3 枚で価値が伝わる構成にする。

### Frame 1 — Before / After（**最重要**）

| 役割 | 内容 |
|---|---|
| ヘッドライン | **植物の時間を、残す。** |
| サブ | 1日目から今日まで。並べるだけで成長が見える。 |
| 端末描画 | `CompareScreen`：上下に Before / After。`1日目 · 2025.01.03` ↔ `411日目 · 2026.05.24` |
| 訴求 | 時間変化・成長比較 |

### Frame 2 — Compare の経過指標

| 役割 | 内容 |
|---|---|
| ヘッドライン | **観測は、積み上がる。** |
| サブ | 経過日数・観測数・水やり回数を一目で。 |
| 端末描画 | `CompareScreen` の Interval カード（411 日 / 観測 N 回 / 水やり 38 回）+ スクラブバー |
| 訴求 | 積み上がり |

### Frame 3 — 観測 Timeline

| 役割 | 内容 |
|---|---|
| ヘッドライン | **日々を、節として残す。** |
| サブ | 撮るたびに Timeline が伸びていく。 |
| 端末描画 | `PlantDetailScreen`：ヒーロー写真 + 観測ストリップ |
| 訴求 | 観測 Timeline |

### Frame 4 — Observation（カメラ）

| 役割 | 内容 |
|---|---|
| ヘッドライン | **構図は、毎回そろう。** |
| サブ | 直前の観測を薄く重ねて定点撮影。 |
| 端末描画 | `CameraScreen`：ダーク全画面 + 植物チップ + ゴーストガイド表示 |
| 訴求 | 観測習慣（定点） |

### Frame 5 — Quick Log

| 役割 | 内容 |
|---|---|
| ヘッドライン | **水やりも、ひと記録。** |
| サブ | 撮らない日も、節として残す。 |
| 端末描画 | `QuickLogScreen`：水やり / 植替え / メモ |
| 訴求 | 育成履歴 |

### Frame 6 — Collection

| 役割 | 内容 |
|---|---|
| ヘッドライン | **コレクションを、静かに。** |
| サブ | 一覧は管理画面ではなく、書架のように。 |
| 端末描画 | `CollectionScreen`：植物グリッド |
| 訴求 | ブランド（Quiet Luxury） |

## 2. レイアウト規定

各フレームは縦長 1 枚で構成。上段にコピー、下段に iPhone デバイス枠を配置する。

```
┌───────────────────────────┐ 1290 × 2796
│  HEADLINE（display）       │  上部 ~720px：背景 void
│  sub text（mono small）    │
│                           │
│   ┌──────────────────┐    │  下部：端末モック（角丸・影なし）
│   │                  │    │  端末はキット 360×780 を 2.55× 拡大して
│   │   iPhone screen   │    │  920×1990 程度に貼る
│   │                  │    │
│   └──────────────────┘    │
│                           │
└───────────────────────────┘
```

- 背景：`var(--c-void)`（純黒寄り）
- ヘッドライン：`var(--font-display)` 88px / weight 300 / letter-spacing -0.02em
- サブ：`var(--font-mono)` 22px / `var(--c-fog)` / letter-spacing 0.06em / uppercase ではなく日本語そのまま
- セーフエリア：左右 80px / 上 96px / 下 96px
- ロゴ：右下に小さく `Node.` のワードマーク（display 32px、`.` を `var(--c-moss)`）

## 3. 制作フロー

1. **HTML 生成** — `design/preview/app-store-v1/index.html` を開くと 6 枚分のフレームが App Store 寸法（1290×2796）でレイアウトされる。既存 UI kit（`design/ui_kits/node-ios/*.jsx`）を直接マウントする。
2. **PNG 出力** — Chrome デベロッパーツールの「ノードのキャプチャ」、または `npx playwright screenshot --viewport-size=1290,2796 …` で 1 枚ずつ書き出す。
3. **6.5" レンディション** — 同じ HTML をビューポート 1242×2688 にしてキャプチャ。スケールは CSS 側で吸収。
4. **コピー検証** — JP 文言は社内（@toshiki.isawa）レビュー後に確定。Apple のガイド（テキストオーバーレイの禁止語：「FREE」「LIMITED」「BEST」等）に該当しないこと。
5. **提出** — App Store Connect の Screenshots セクションに 6.9" → 6.5" の順で 6 枚アップロード。

## 4. 受け入れ基準

- [ ] 6 枚すべて 1290 × 2796（6.9"）で書き出されている
- [ ] 同 6 枚を 1242 × 2688（6.5"）でも書き出している
- [ ] Search / Tag / iCloud / 同期 / 課金 が画面・コピーいずれにも出ていない（Timelapse は v1.0 無料のため可）
- [ ] Before / After フレームが最初（または 2 枚目までの中）に配置されている
- [ ] ヘッドライン・サブが画面外に見切れていない（セーフエリア 80px 確保）
- [ ] 端末モックの角丸・ホームインジケーター位置が `ios-frame.jsx` のデフォルトと一致している
- [ ] 6.5" 版も同等の見栄え（コピーが折り返しで崩れていないこと）
