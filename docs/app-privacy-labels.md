# App Store Privacy Nutrition Labels 申告ガイド

Node. iOS（`app.node.ios`）を App Store Connect に登録する際の **App Privacy** 申告のたたき台です。  
実際の申告は [Apple のデータ型定義](https://developer.apple.com/app-store/app-privacy-details/) および各 SDK の最新ドキュメントに合わせて最終確認してください。

---

## 0. リリース段階

| 段階 | `ReleaseConfig` | App Store 申告 |
|------|-----------------|----------------|
| **v1.0（現行・App Store 初版）** | `cloudSyncEnabled = false` / `timelapseEnabled = false` | **使用状況データ（匿名）のみ収集**（PostHog · オプトアウト可）。観測データは端末内のみ |
| **v1.0.1（Node. Pack 予定）** | `timelapseEnabled = true` | 非消費型 IAP（StoreKit）・CloudKit プライベート DB を追加申告 |
| **v1.1（予定）** | 両方 `true` | 有料クラウド同期・StoreKit サブスクを追加申告 |

v1.0 では Supabase / StoreKit の SDK は同梱されていても **呼び出されません**。PostHog は v1.0 から有効化し、匿名のアプリ内行動イベント（画面遷移・操作回数など）のみを収集する（個人情報・写真・植物名は送信しない、ATT 許可不要）。Settings からオプトアウト可能。申告は **v1.0 の実際の挙動** に合わせてください。

---

## 1. v1.0 申告（App Store 初版）

### 1.1 データ収集の有無

| 質問 | 回答 |
|------|------|
| あなたまたは第三者パートナーはこのアプリからデータを収集しますか？ | **はい**（使用状況データのみ） |

v1.0 では観測記録（植物・写真・メモ）は端末内（SwiftData）のみで、当方サーバーには送信されません。一方で、PostHog（first-party 分析）でアプリ改善のための匿名行動イベント（画面遷移、ボタンタップ回数、通知許可状態など）を収集します。Settings からいつでも送信を停止できます。

### 1.2 収集データの分類

#### 使用状況データ

| データ型 | 収集 | リンク | トラッキング | 用途 |
|----------|------|--------|--------------|------|
| 製品の操作 | はい | いいえ | いいえ | アプリ機能 / 製品の個人化 |
| その他の使用状況データ | はい | いいえ | いいえ | アプリ機能 |

- **リンク**: 当方は個人を特定する識別子を保持していないため、ユーザーへのリンクなし
- **トラッキング**: PostHog は first-party 分析として扱う。ATT 許可は求めない
- イベントには `app_version`, `os_version`, `device_model` 等の端末メタ情報を含むが、植物名 / 写真 / メモ / 位置情報は **一切送信しない**

#### 診断

| データ型 | 収集 | リンク | トラッキング | 用途 |
|----------|------|--------|--------------|------|
| クラッシュデータ | （任意で追加） | いいえ | いいえ | アプリ機能 |

PostHog 経由でフロントエンドのエラーログを送る場合に申告。送らない場合は不要。

### 1.3 トラッキング

| 質問 | 回答 |
|------|------|
| トラッキングのためにデータを使用しますか？ | **いいえ** |

ATT 許可は求めない。PostHog の distinct_id は端末ローカルの UUID で、他社サービスやプロファイルに紐づけない。

### 1.4 端末権限（参考）

App Privacy とは別に、Info.plist で以下を申告・説明します。

| 権限 | 用途 |
|------|------|
| カメラ | 観測写真の撮影 |
| 写真ライブラリ | 観測写真・タイムラプス動画の保存 |
| 通知 | 水やりリマインダー（ユーザーが Settings から ON にした場合のみ） |

### 1.5 v1.0 チェックリスト

- [ ] App Privacy: **使用状況データを収集**（リンクなし・トラッキングなし）を選択
- [x] `docs/privacy-policy.md` 第 2 条（運営者・連絡先）を記入
- [x] v1.0 専用プライバシーポリシー（クラウド・課金の条項なし。PostHog 利用と Settings オプトアウトの節を追加）
- [x] アプリ内プライバシーポリシー — 同梱 `ios/Node/Resources/privacy.html`（`PRIVACY_POLICY_URL` 未設定）
- [ ] App Store Connect — プライバシーポリシー URL に以下を登録
- [x] 公開 URL 取得 — [プライバシーポリシー](https://node-app.jp/privacy/)
- [ ] App Store 説明文に「記録は端末内保存。機種変更時は引き継げません」を明記
- [ ] PostHog プロジェクトのデータ保持期間と GDPR / CCPA 設定を確認

## 2. v1.0.1 申告（Node. Pack）

v1.0.1 で Node. Pack（買い切り IAP）と CloudKit バックアップを導入する。詳細は [docs/v1.0.1-pack-plan.md](./v1.0.1-pack-plan.md)。

### 2.0 データ収集の有無

| 質問 | 回答 |
|------|------|
| あなたまたは第三者パートナーはこのアプリからデータを収集しますか？ | **はい**（Pack 購入時のみ） |

- StoreKit: トランザクション ID（Pack 購入時）
- CloudKit: 観測写真・メタデータをユーザー個人の iCloud プライベート DB に保管（Apple サーバー。当方サーバーには送信しない）

## 3. v1.1 申告（有料クラウド同期・予定）

v1.1 リリース時に `ReleaseConfig` を有効化したうえで、以下を追加申告します。

### 3.1 データ収集の有無

| 質問 | 回答 |
|------|------|
| あなたまたは第三者パートナーはこのアプリからデータを収集しますか？ | **はい** |

Archive / Conservatory 利用時に Supabase / R2 へデータが送信されます。

### 3.2 収集データの分類（たたき台）

#### 連絡先情報

| データ型 | 収集 | リンク | トラッキング | 用途 |
|----------|------|--------|--------------|------|
| メールアドレス | 場合あり | はい（アカウント） | いいえ | アプリ機能、認証 |

Sign in with Apple / Google 経由。有料プラン・サインイン時のみ。

#### ユーザーコンテンツ

| データ型 | 収集 | リンク | トラッキング | 用途 |
|----------|------|--------|--------------|------|
| 写真または動画 | はい | はい | いいえ | アプリ機能 |
| その他ユーザーコンテンツ | はい | はい | いいえ | アプリ機能 |

有料プラン同期時。タイムラプス MP4 は端末内生成のみ（クラウド非保存）。

#### 識別子

| データ型 | 収集 | リンク | トラッキング | 用途 |
|----------|------|--------|--------------|------|
| ユーザー ID | はい | はい | いいえ | アプリ機能、分析 |

#### 購入

| データ型 | 収集 | リンク | トラッキング | 用途 |
|----------|------|--------|--------------|------|
| 購入履歴 | はい | はい | いいえ | アプリ機能 |

StoreKit トランザクション ID。Pack 購入 / 有料プラン購入時。

#### 使用状況データ・診断

PostHog 有効時: アプリ内挙動の集計のみ。トラッキング用途には該当しない想定。

### 3.3 トラッキング

| 質問 | 回答 |
|------|------|
| トラッキングのためにデータを使用しますか？ | **いいえ** |

ATT 許可は求めない。

## 5. プライバシーポリシー URL

| 用途 | URL / ファイル |
|------|------------------|
| **App Store Connect 申告** | https://node-app.jp/privacy/ |
| **v1.0 アプリ内表示** | [ios/Node/Resources/privacy.html](../ios/Node/Resources/privacy.html)（`PRIVACY_POLICY_URL` 未設定時） |
| **リポジトリ正本** | [docs/privacy-policy.md](./privacy-policy.md) |
| **リポジトリ内 Web 版** | [web/privacy.html](../web/privacy.html)（Notion と内容を揃える参考用） |

v1.0 では Settings から同梱 HTML を表示する。App Store Connect には上記公開 URL を登録する。公開ページの本文は v1.0 専用ポリシー（端末内のみ・サーバー送信なし）と一致させること。
