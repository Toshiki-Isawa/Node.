### 1.8 Compare 表示モード（スライダー + 並列・2 択）

**目的**: コアユーザーが「境界を動かす実感」と「2 枚を同時に検証する分析」を切り替えられるようにする。モード数は **2 択のみ**（スライダー / 並列）。左右・上下の軸選択はユーザーに露出せず、端末向きで自動決定する。

**リリース**: **v1.0.1**（Seed 標準・Pack ゲートなし）。v1.0 提出前に実装余力があれば前倒し可。詳細設計は [specification.md §10.6](../specification.md) 参照。

#### 1.8.1 設計サマリ

| 項目 | 決定 |
|------|------|
| モード数 | **2 のみ** — `slider`（デフォルト）/ `split`（並列） |
| デフォルト | **スライダー**（初回起動・再インストール後も同じ） |
| 並列の軸 | ユーザー選択なし。**縦持ち → 上下分割**、**横持ち → 左右分割** |
| 並列の上下左右の意味 | 上（または左）= **Before**、下（または右）= **After** |
| 課金 | Seed / Pack / Sub すべて **無料**（機能ゲート対象外） |
| 永続化 | 最後に選んだモードを `UserDefaults` に保存（次回 Compare 表示時に復元） |

#### 1.8.2 UX 仕様

**配置（比較画像エリア右上）**

- セグメント風の **アイコン 2 ボタン**（テキストラベルなし）
  - スライダー: `slider.horizontal.below.rectangle`（または現行に近い SF Symbol）
  - 並列: `rectangle.split.2x1`（縦）/ 向きに応じて `rectangle.split.1x2`（横）は **アイコンは 1 種固定**でよい（システムが向きで見た目が変わるため）
- 比較画像の `ZStack` 右上、日付オーバーレイと重ならないよう `12pt` パディング + `.ultraThinMaterial` 背景のピル型コンテナ
- **アクセシビリティ**: `accessibilityLabel` — 「スライダーで比較」「並列で比較」

**スライダーモード（現行踏襲）**

- 既存 [`ImageComparisonSlider`](ios/Node/Components/ImageComparisonSlider.swift) をそのまま使用
- ドラッグで Before / After の境界を移動（デフォルト位置 50%）
- BEFORE / AFTER ラベル、日付タップ → カレンダーシートは **現行どおり**

**並列モード（新規）**

- 新規コンポーネント `ImageComparisonSplit`（`ios/Node/Components/`）
- 同一 `aspectRatio`（4:3）の枠内を `GeometryReader` で分割
  - `verticalSizeClass == .compact` または幅 > 高 → **HStack**（左 Before / 右 After）
  - それ以外（縦持ち）→ **VStack**（上 Before / 下 After）
- 各ペイン: `scaledToFill` + `clipped`（スライダーと同じ画像読み込み・プレースホルダー）
- 分割線: `NodeColor.bone` 1〜2pt（スライダー仕切りとトーン統一）

- 並列では **ピンチズーム同期は v1.0.1 スコープ外**（将来 §10.11 比較画像 Export とセットで検討）

## 6. リスク・未解決事項

### 6.1 既知のリスク

- **CloudKit 本番運用が初**: スキーマミスや想定外エラーは出る前提。v1.0.1 で実利用者の挙動を観測しながら v1.1 R2 同期の前に枯らす。
- **Family Sharing 不可逆**: 一度 ON にすると OFF できない。再度確認推奨。
- **Notification Service Extension のメモリ制限**: 24MB 上限。SwiftData 共有コンテナでの CareLog 追加が収まるか実機で要検証。
- **iCloud 容量推定不可**: Apple は残容量 API を出していない。Reactive な OOQ ハンドリングに依存。

### 6.2 後で決める

- 「Node. Pack」の正式商品名（Conservator's Pack / Botanical Pack / Archive Pack 等の世界観統一）
- 値上げタイミングの具体日（ローンチから 6 ヶ月 / 1 年）
- プロモコード配布枠（年 100 枠）の使い方
- Pack 紹介スクリーンショットのデザイン
- 「最近削除した項目」ローカルゴミ箱の v1.0.1 以降の検討（Q17 で v1.0.1 範囲外と決定）
