## 1. 概要

> **育てた記憶まで、残していく。**
> 植物の時間を残す観測アーカイブ。

### 1.1 プロダクト概要

Node. は「植物の時間を残す」をコンセプトとした、植物観測アーカイブアプリである。植物の観測・成長・育成履歴を蓄積し、定点観測・成長比較・タイムライン・タイムラプス・育成履歴・観測習慣を通じて、「植物を観測する体験そのもの」を価値化する。

### 1.2 プロダクトの本質

Node. は「植物を管理するアプリ」ではなく、「植物の時間を記録するアプリ」である。最終的には **植物観測 OS** として進化し、**植物の時間データ基盤** となることを目指す。

### 1.3 最重要思想

Node. は「植物を育てるアプリ」ではなく、**「植物の時間を残すアプリ」** である。

---

## 2. プロダクト思想

### 2.1 Observation First

Node. の中心は **Observation（観測）** である。すべての機能設計・UX 判断は、観測体験の質を最優先とする。

### 2.2 提供価値

| 価値 | 説明 |
|------|------|
| 成長記録 | 植物の成長を継続的に記録できる |
| 時間変化の比較 | 任意の時点間で変化を視覚的に比較できる |
| 育成履歴の蓄積 | 観測データが時系列で積み上がり、資産となる |
| 観測習慣の形成 | 定点観測が自然な習慣として定着する |
| 時間の資産化 | 植物固有の「時間」が長期的な価値を持つ |

### 2.3 競合との差別化

| 一般的な植物アプリ | Node. |
|-------------------|-------|
| 水やり通知中心 | 観測中心 |
| AI 診断単発 | 長期履歴蓄積 |
| 管理ツール | 観測アーカイブ |
| SNS 投稿 | 個人記録 |
| Todo 感 | ギャラリー感 |

Node. は **AI 診断アプリ・SNS・水やり通知アプリ** ではない。差別化の核は **観測履歴** である。

|----------|------|

### 8.2 画像・Archive 戦略

Node. の価値は **「植物の時間を積み上げること」** にある。課金は機能ではなく **保存品質・Archive・Original・長期履歴** を中心に設計する（12.2 参照）。

| 保存先 | 役割 | 方針 |
|--------|------|------|
| ローカル（Original） | 撮影直後の即時保存 | v1.0: 端末内に保持。v1.1 有料同期後は R2 正本へ退避し端末から削除 |
| ローカル（サムネイル） | オフライン閲覧 | 常駐。Timeline / Collection 表示に使用 |
| クラウド（R2） | 長期 Archive 正本 | **v1.1 以降・有料プランのみ**。Archive / Conservatory: Original |
| ローカル（キャッシュ） | 一時取得 | v1.1 有料同期後、Compare / Timelapse 利用時に R2 からオンデマンド取得 |
| タイムラプス MP4 | 端末内の一時生成物 | **保存しない**（Export 先はユーザー任意。インフラコスト回避） |

| プラン | ローカル Original | ローカルサムネイル | クラウド保存 | クラウド容量 |
|--------|-------------------|-------------------|--------------|--------------|
| Seed | 端末内に保持 | 常駐 | ×（v1.0〜） | — |
| Archive | v1.1: 同期完了まで保持 | 常駐 | Original 保持 | 50 GB（調整可能） |
| Conservatory | v1.1: 同期完了まで保持 | 常駐 | Original 保持 | 500 GB（調整可能） |

**リリース段階:** v1.0（App Store 初版）は Seed を含め **端末内のみ**。クラウド同期・課金は v1.1 で Archive / Conservatory 向けに提供する（`ReleaseConfig` 参照、[ios/README.md](ios/README.md)）。

詳細は 11.5（クラウド同期・Archive）・12.2（プラン構成）を参照。

### 10.3 Camera UX

**最重要機能。** Node. の価値の中心。

#### 要件

| 要件 | 詳細 |
|------|------|
| 画面向き | 縦固定 |
| 起動速度 | カメラを素早く起動 |
| 撮影テンポ | 連続撮影のテンポを損なわない |

| オフライン対応 | ネットワーク未接続でも撮影・保存可能 |

#### 撮影フロー

```
1. 植物選択
2. カメラ起動
3. 撮影
4. 即保存
5. （任意）ログ追加
```

- ローディングを見せない
- 通信待ちさせない
- 撮影テンポを止めない
- ログ入力を強制しない
- 保存を即時完了に感じさせる

#### カメラ UI 思想

- 植物が主役
- UI は最小限
- 撮影行為に集中

#### 撮影補助機能

- グリッド
- 前回写真オーバーレイ
- 定点補助

### 10.6 Comparison（比較）

Node. の価値を最も感じやすい機能。

#### 要件

- Before / After 形式
- 任意の 2 観測点間での比較（カレンダーから Before / After を個別に選択）
- 日数表示（観測日目・経過日数・観測差・期間中の水やり回数）
- **表示モードは 2 択のみ**（ユーザーが選ぶのはモードだけ。左右 / 上下の軸は選ばせない）

| モード | 役割 | デフォルト |
|--------|------|------------|
| **スライダー** | 同じ構図の定点観測で、境界を動かして変化を「実感」する | **○** |
| **並列** | 2 枚を同時に見て、構図のズレや量的変化（葉数・鉢サイズ等）を検証する | — |

#### 表示モード詳細

**スライダー**

- 横方向ドラッグで Before / After の境界を移動（初期位置 50%）
- 実装: `ImageComparisonSlider`（既存）

**並列**

- 1 画面に Before と After を同時表示（各画像は常にフル表示領域内）
- 分割軸は **端末向きで自動**
  - 縦持ち（portrait）: **上下分割** — 上 = Before、下 = After
  - 横持ち（landscape）: **左右分割** — 左 = Before、右 = After
- 実装: `ImageComparisonSplit`（v1.0.1 で追加。計画 [docs/v1.0.1-pack-plan.md](docs/v1.0.1-pack-plan.md) §1.8）

**共通 UI**

- 比較画像エリア右上: アイコン 2 つでモード切替（スライダー / 並列）
- 各モードで日付・日数目タップ → 観測日カレンダーシート（現行どおり）
- 最後に選択したモードを端末に記憶（次回 Compare 表示時に復元）

**課金・リリース**

- Seed / Pack / Sub いずれも **同一 UX**（機能ゲートなし）
- v1.0.1 で提供（v1.0 提出前の前倒しは任意）

**v1.0.1 スコープ外（将来）**

- 並列モードのピンチズーム同期
- 比較画像 Export 時のスライダー中間位置キャプチャ（Export 実装時は並列レイアウトをデフォルト推奨 — §10.11）

### 10.7 Timelapse（タイムラプス）

#### リリース方針

タイムラプス動画生成は **v1.0 から全ユーザーに無料提供** する（`ReleaseConfig.timelapseEnabled = true`）。当初は v1.0.1 の Node. Pack 専売機能とする方針だったが、観測体験の価値を広く届けるため無料開放へ変更した。画質は既存のプラン別ティア（無料 Seed = 720p / 有料 Archive・Conservatory = 4K）を維持する。実装本体は `TimelapseService` / `TimelapseVideoGenerator` / `TimelapseView`。

**価値の根拠：**

- 価値が一瞬で伝わる
- SNS 共有と相性が良い
- 継続モチベーションになる

#### 基本方針（インフラ・保存）

| 項目 | 方針 |
|------|------|
| 生成場所 | **端末内のみ**（`AVFoundation` 等） |
| 入力素材 | ローカルに保存済みの観測画像（Original / サムネイル） |
| クラウド保存 | **行わない**（R2・Supabase へ動画をアップロードしない） |
| 恒久保存 | アプリ内 DB には保持しない。Export 時に写真ライブラリ保存または共有シートでユーザーが任意保存 |
| サーバー生成 | **採用しない**（FFmpeg ワーカー・Edge Function による生成は廃止方針） |

インフラコスト増大を避け、観測画像（既存の Local-first 資産）だけで成果物を作る。

#### 出力形式

| 項目 | 方針 |
|------|------|
| コンテナ | MP4（H.264 等、端末がサポートするコーデック） |
| Export 品質 | 無料 Seed = 720p / 有料 = 最大 4K（端末性能・ソース解像度の範囲内） |
| アスペクト | 9:16 縦長（Instagram / TikTok 向け） |

#### 技術方針（iOS）

| 項目 | 方針 |
|------|------|
| フレームソース | 選択した Observation のローカル画像を時系列で並べる |
| 最小枚数 | 5 枚以上（`TimelapseRequirements.minimumObservations`） |
| 最大フレーム数 | 60 枚（超過分は間引きまたは末尾優先で採用） |
| 処理 | バックグラウンドでエンコード。UI は進捗表示 |
| 完了後 | プレビュー → Export（共有シート / 写真ライブラリ） |
| ネットワーク | 生成・Export に **必須ではない**（オフライン観測データのみで完結可能） |

#### 非要件（明示的にやらないこと）

- タイムラプス動画のクラウド同期・アーカイブ
- サーバーサイドのタイムラプス生成 API・ジョブテーブル
- 生成動画の Supabase / R2 上の長期保管
- 機種変更時のタイムラプス動画のクラウド復元

#### プラン別の品質

- v1.0 以降・全プラン：エントリポイントを表示し、端末内生成・Export を無料提供
- Seed（無料）：720p Export
- Archive / Conservatory（有料）：4K Export

#### 受け入れ基準

- v1.0：Compare 画面に「タイムラプス」ボタンが表示される（`ReleaseConfig.timelapseEnabled == true`）

## 11. 非機能要件

### 11.1 パフォーマンス

- カメラ起動は体感待ち時間を最小化すること
- 撮影後の保存処理はユーザー操作をブロックしないこと
- 保存を即時完了に感じさせること

### 11.2 オフライン / 同期（Local First）

#### 方針

Observation は **ローカル保存優先** とする。

| 方式 | 詳細 |
|------|------|
| Background Sync | バックグラウンドで同期 |
| Queue 方式 | 同期キューによる順次処理 |
| 非同期アップロード | ユーザー操作をブロックしない |

#### UX 思想

**「通信を意識させない」**

Local First は **端末への永久フル保存** を意味しない。撮影後は即ローカル Original を保存し、**同期完了後は R2 を正本として Original を端末から退避** する。サムネイルは端末に常駐し、Compare / Timelapse 等で Original が必要な場合は R2 からオンデマンド取得する。クラウド同期・容量管理の詳細は 11.5 を参照。

### 11.3 可用性

- オフライン環境でも観測・撮影・Quick Log が利用可能
- 同期失敗時もローカルデータは保持され、再試行可能

### 11.4 セキュリティ

- ユーザーデータは認証済みユーザーのみがアクセス可能
- 画像データは適切なアクセス制御のもと Cloudflare R2 に保存

### 11.5 クラウド同期・Archive

#### 基本思想

Node. は長期利用によって価値が増すプロダクトである。**v1.0 では Seed 含め端末内保存のみ** とし、集客・観測習慣形成に集中する。**v1.1 以降**、Archive / Conservatory 向けにクラウド同期を提供し、**「植物の時間を残す」** 価値を有料化する。

| 原則 | 説明 |
|------|------|
| v1.0 ローカル無料 | サインイン・同期・課金 UI を非表示。インフラコスト $0 |
| 有料のみクラウド | v1.1 以降、R2 同期は Archive / Conservatory のみ（Seed は端末内のみ） |
| 履歴は消さない | 一定期間での履歴削除・古い Observation の消去は行わない |
| 差別化軸 | 履歴期間ではなく **保存品質**・**保存容量**・**クラウドバックアップ** で差別化 |
| 観測体験優先 | 容量超過でも撮影・ローカル保存は継続。同期のみ停止 |
| クラウド正本 | v1.1 有料同期後、長期 Archive の正本は R2 とする |
| 端末 Original 退避 | 有料同期成功後、Original を端末から削除。サムネイルは保持 |
| オンデマンド取得 | Compare / Timelapse 等で Original が必要な場合、R2 から presigned URL で一時取得 |

#### リリース段階（ReleaseConfig）

| 段階 | cloudSyncEnabled | subscriptionsEnabled | 内容 |
|------|------------------|----------------------|------|
| v1.0 | `false` | `false` | 端末内観測フローのみ。Supabase / R2 本番不要 |
| v1.1 | `true` | `true` | 有料プランでクラウド同期・StoreKit 有効。Seed はローカルのみ |

実装: `ios/Node/Core/Config/ReleaseConfig.swift`

#### 同期戦略（v1.1 以降）

| 方式 | 詳細 |
|------|------|
| Local First | 撮影後は即ローカル保存。同期はバックグラウンド実行 |
| 有料のみ同期 | Archive / Conservatory サインイン時のみ SyncEngine が動作 |
| Queue 方式 | 同期キューによる順次処理 |
| 非同期アップロード | ユーザー操作をブロックしない |
| バックフィル | v1.0 利用者は v1.1 購入・サインイン後にローカルデータを一括同期 |

#### Seed — クラウド保存

| 項目 | 方針 |
|------|------|
| v1.0 | クラウド同期なし。端末内のみ |
| v1.1 以降 | クラウド同期なし（無料プランは端末内 Archive） |
| 履歴保持期間 | 無制限（端末内） |

#### Archive — クラウド保存

| 項目 | 方針 |
|------|------|
| 保存品質 | Original（HEIF / 高解像度） |
| 容量上限 | 50 GB（調整可能） |
| タイムラプス動画 | クラウド保存しない（端末内生成・ユーザー Export のみ） |
| 復元 | 観測画像・メタデータの完全復元（タイムラプス MP4 は対象外） |

#### Conservatory — クラウド保存

| 項目 | 方針 |
|------|------|
| 保存品質 | Original |
| 容量上限 | 500 GB（調整可能） |
| 将来価値 | RAW 保持、Lineage、Ownership History |

#### 容量上限到達時

| 動作 | 詳細 |
|------|------|
| 新規 Observation | ローカル保存継続 |
| クラウド同期 | 停止（`sync_paused_storage_limit`） |
| アプリ利用 | 継続可能 |
| 上位プラン導線 | Archive / Conservatory への静かな案内 |

#### Seed クラウド圧縮品質

v1.1 以前の Seed 3 GB 圧縮同期案は **v1.0 ローカル無料戦略により廃止**。以下は将来 Seed にクラウドを追加する場合の参考値。

| 項目 | 方針 |
|------|------|
| 最大辺 | 2048 px |
| JPEG 品質 | 0.72 前後 |
| 目的 | Compare / Timelapse に十分な画質を確保しつつ容量を抑制 |

#### UX 原則（禁止事項）

- 同期失敗による保存ブロック
- 容量超過による撮影停止

#### 許容する挙動

- 同期のみ停止
- ローカル保存継続
- 後から Archive / Conservatory へ移行可能

#### 将来拡張（Post-MVP）

**Smart Compression（Free 向け）**

| 期間 | 品質 |
|------|------|
| 直近 30 日 | 高品質 |
| 30 日以降 | 高圧縮 |

**将来の Premium 価値:** RAW 保存、Plant Passport、Lineage 履歴、Transfer 完全同期、AI 履歴分析

## 12. マネタイズ戦略

### 12.1 基本思想

Node. のマネタイズは **観測体験を阻害しない** ことを最優先とする。Node. は **機能課金** ではなく、**時間の蓄積** — 保存品質・Archive・Original・長期履歴 — を中心に課金価値を設計する。

| プラン | コンセプト | 方針 |
|--------|-----------|------|
| Seed | 観測を始める | v1.0: 端末内無制限・タイムラプス無料（720p）。v1.1: 引き続き端末内のみ |
| Pack | 植物の時間を残す道具 | v1.0.1: 買い切り IAP。iCloud バックアップ・全文検索・タグ（タイムラプスは v1.0 から無料のため Pack 対象外） |
| Archive | 植物の時間を残す | v1.1: Original クラウド同期・50 GB・Export |
| Conservatory | コレクションを保存する | v1.1: Archive 全機能 + 500 GB + 将来のコレクター機能 |

マネタイズは **保存品質・Archive・Original・長期履歴 + 道具としての Pack** で構成する。Pack の詳細は [docs/v1.0.1-pack-plan.md](docs/v1.0.1-pack-plan.md) §0.2 / §0.3。

### 12.2 プラン構成（松竹梅）

#### Seed（Free）

**コンセプト:** 観測を始める

| 機能 | 内容 |
|------|------|
| Plant / Observation | 無制限 |
| Timeline / Compare / Quick Log | ○ |
| ローカル保存 | Original + サムネイルを端末内に保持（v1.0 デフォルト） |
| クラウド同期 | ×（v1.0〜。有料プランは v1.1 で提供） |
| 履歴保持期間 | 無制限（端末内） |
| クラウド容量 | — |
| Timelapse 生成（端末内） | ○（720p・v1.0 から無料） |

**UX 方針:** 観測体験は制限しない。v1.0 では「記録はこの端末に保存されます」と明示する。タイムラプスは v1.0 から全プラン無料（Seed は 720p）。機種変更時のデータ引き継ぎは v1.0.1 Pack の iCloud バックアップで提供。

#### Conservatory（Collector · 月額 980〜1480 円想定）
